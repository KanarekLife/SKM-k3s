Czym jest kubernetes?

  • Kubernetes jest system open-source napisanym w Go do automatyzacji wdrożeń, skalowania oraz zarządzania skonteneryzowanymi aplikacjami.
  • Został stworzony w 2014 przez programistów w Google'u zainspirowanych wewnętrznym narzędziem "Borg" służącym do zarządzania klastrami oraz "Promise Theory".
  • Wersja 1.0 została wydana w 2015 i została przekazana do zarządzania nowo-utworzonej organizacji "Cloud Native Computing Foundation (CNCF)" utworzonej przez Google oraz Linux Foundation. Do tej pory odpowiadają za popularyzację, rozwój i utrzymanie kubernetes'a i wielu innych powiązanych projektów.
  • Kubernetes operuje w sposób deklaratywny na obiektach opisujących pożądany stan działających wdrożeń na klastrze. Sam system napisany jest w sposób ułatwiający jego rozbudowę oraz umożliwia duże możliwości zarówno dla developerów, operatorów jak i administratorów klastrów.

Najczęstsze sposoby wdrażania aplikacji

  • Wdrożenie na system operacyjny (np. instalacja .NET Runtime i `dotnet run app.dll`)
  • Wdrożenie na maszynę wirtualną
  • Wdrożenie na kontenery z systemem operacyjnym, np. BSD jails, LXC
  • Wdrożenie na kontenery, np. docker, podman
  • Wdrożenie na klaster kubernetes
  • Wdrożenie na zarządzaną usługę w chmurze, np. Azure WebApp Service, Google Cloud Run etc.

Pets - usługi którymi "zajmujemy się" i utrzymujemy, Cattle - usługi które możemy łatwo tworzyć i niszczyć według potrzeb

Parę największych plusów i minusów kubernetes'a

    Plusy

  • Ustandaryzowany sposób wdrażania aplikacji; redukujemy ryzyko vendor lock-in'u
  • Szereg feature'ów ułatwiających pracę: automatycznie leczenie zdegradowanych usług, automatyczne skalowanie usług w zależności od obciążenia, możliwość łatwiejszego zbierania statystyk...
  • Możliwość delegacji uprawnień, dzięki czemu różne zespoły mogą mieć dostępy do różnych usług działających na tym samych klastrze / węźle co poprawia wykorzystanie aktualnych zasobów
  • Dużo materiałów do nauki i projektów korzystających z k8s, dzięki czemu łatwiej się go nauczyć niż mniej popularnych rozwiązań
  • Dzięki deklaratywnej naturze, łatwiej skalować nasze aplikacje

    Minusy

  • Wyższe koszty (zwłaszcza jeżeli chcemy skorzystać z cloud'owych rozwiązań albo mieć rozproszony klaster)
  • Administracja i korzystanie wymaga dosyć sporej wiedzy
  • Może być trudniejszy do debugowania w razie problemów
  • Może nie być opcją gdy nasza aplikacja nie jest skonteneryzowana

Z czego zbudowany jest klaster kubernetes?

  • Node

    • kubelet - An agent that runs on each node in the cluster. It makes sure that containers are running in a pod.
    • Container Runtime - The container runtime is the software that is responsible for running containers.
    • kube-proxy - a network proxy that runs on each node in the cluster.

  • Control Plane

    • ectd - a consistent and highly-available key value store used as Kubernetes' backing store for all cluster data.
    • kube-api-server - The API server is a component of the Kubernetes control plane that exposes the Kubernetes API. The API server is the front end for the Kubernetes control plane.
    • scheduler - Control plane component that watches for newly created Pods with no assigned node, and selects a node for them to run on.
    • Controller Manager - Control plane component that runs controller processes. (Node controller: Responsible for noticing and responding when nodes go down, Job controller: Watches for Job objects that represent one-off tasks, then creates Pods to run those tasks to completion, EndpointSlice controller: Populates EndpointSlice objects (to provide a link between Services and Pods), ServiceAccount controller: Create default ServiceAccounts for new namespaces)
    • cloud-control-manager - A Kubernetes control plane component that embeds cloud-specific control logic. If you are running Kubernetes on your own premises, or in a learning environment inside your own PC, the cluster does not have a cloud controller manager.

Na czym postawić klaster do nauki?

  • AKS, EKS, GKE - bardzo dobre na produkcyjny workload, aczkolwiek mogą mieć zbyt wysoką cenę do nauki lub środowiska dev/test. Zawierają elementy spoza domyślnych elementów kubernetes'a, także mogą występować pewne różnice (np. load balancer).
